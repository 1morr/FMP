## 2. LyricsAutoMatchService: 提取重复的匹配保存逻辑

**文件:** `lib/services/lyrics/lyrics_auto_match_service.dart`

**问题:** `tryAutoMatch()` 中的匹配保存逻辑（缓存写入 + LyricsMatch 创建 + 数据库保存）在 4 个地方重复出现（直接获取 + netease/qqmusic/lrclib 搜索匹配）。

**修复:**
- 提取 `_saveMatch()` 辅助方法，统一处理缓存和数据库保存
- 简化 for 循环中的 switch 逻辑，将结果统一赋值给 `result` 变量后统一处理
- 减少约 30 行重复代码

## 3. DownloadService: 修正误导性的线程安全注释

**文件:** `lib/services/download/download_service.dart`

**问题:** `_recordProgressUpdate()` 的注释声称 "Dart 的 Map 操作是原子的" 来解释线程安全性。这是不准确的——Dart 的线程安全来自单 Isolate 事件循环模型，而非 Map 操作的原子性。同时注释说回调在 "IO 线程中" 调用，实际上 Dio 的回调在主 Isolate 事件循环中执行。

**修复:** 更正注释，准确描述 Dart 的并发模型。

## 4. AudioController: 位置检测常量移至 AppConstants

**文件:** `lib/services/audio/audio_provider.dart`, `lib/core/constants/app_constants.dart`

**问题:** `_positionCheckInterval` 和 `_positionThreshold` 作为私有常量定义在 `AudioController` 内部，与项目中其他时间相关常量集中在 `AppConstants` 的惯例不一致。

**修复:**
- 在 `AppConstants` 中添加 `positionCheckInterval` 和 `positionCheckThreshold`
- 更新 `AudioController` 中的引用

## 5. SourceManager: 添加 dispose 方法关闭 Dio 实例

**文件:** `lib/data/sources/source_provider.dart`

**问题:** `BilibiliSource` 和 `YouTubeSource` 各自创建了 Dio HTTP 客户端实例并提供了 `dispose()` 方法，但 `SourceManager` 从未调用这些 dispose 方法。Provider 也没有注册 onDispose 回调。

**修复:**
- 在 `SourceManager` 中添加 `dispose()` 方法，遍历并关闭所有音源的资源
- 在 `sourceManagerProvider` 中注册 `ref.onDispose()` 回调
